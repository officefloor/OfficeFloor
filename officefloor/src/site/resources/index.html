OfficeFloor

OfficeFloor is inversion of coupling control that enables building applications via First-Class Procedures

In other words, local microservices

Dependency Injection was an evolution in the way objects are managed.

OfficeFloor is another evolution with:

  • Continuation Injection managing the functions
  • Thread Injection managing the threads

These new dimensions do not mean more complexity in your code.

The more complete paradigm means less "work around" code you need to manage. Plus new capabilities are introduced such as complete graphical configuration of your applications.

Read more ...

OfficeFloor is a fundamental change to blend:

  • functional and object-oriented techniques
  • blocking and non-blocking I/O patterns
  • a host of new more complete concepts

into a unified First-Class Procedure model.

First-Class Procedures give you all the benefits of microservices, without the heavy overheads!

OfficeFloor measures performance in two key aspects

  • rapid application development and ease of changes (see tutorials)
  • runtime performance (see comparisons)

OfficeFloor enables faster building of faster applications.
