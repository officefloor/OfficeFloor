Thank you for taking interest in OfficeFloor.

Please follow these steps to start using OfficeFloor.

Step 1: ensure your Java installation has JavaFx modules installed. See JavaFx for more details.

From Java 11 many modules (including JavaFx) are no longer bundled with a Java installation.

Also a minimum version of Java 8 is required due to JavaFx.

Step 2: ensure Eclipse version is Oxygen or higher. Ideally Eclipse Photon

For older Java versions on linux the following environmental variable may be required to avoid crashes in not supporting the later GTK3: export SWT_GTK3=0

For newer versions of Eclipse (2018-09 onwards), GTK2 is no longer supported (requires GTK3). Java8, however, only supports GTK2. With many cloud providers not yet supporting Java higher than 8, this typically means Eclipse Photon is latest version of Eclipse to use.

Step 3: drag the button to your Eclipse IDE to install e(fx)clipse

Should this not find the E(fx)clipse IDE feature, then please follow steps at e(fx)clipse tooling to install

Step 4: drag the button to your Eclipse IDE to install OfficeFloor

Should this not find the OfficeFloor features, please manually install with following steps:

Within your Eclipse IDE go to install new software (typically Help -> Install New Software ...)

Add the OfficeFloor update site: http://officefloor.net/update

Check OfficeFloor and install

Step 5: build your application with OfficeFloor

More information is available in the tutorials to start developing with OfficeFloor
